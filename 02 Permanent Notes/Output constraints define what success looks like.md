# Output constraints define what success looks like

[Certain] Output constraints make the expected result more concrete.

Constraints can specify format, length, language, tone, section structure, code language, design requirements, or exclusions.

Without constraints, an AI response may be correct but still unusable because it does not match the user's actual situation.

Examples of useful constraints:

- "Use Markdown."
- "Give three options."
- "Keep the answer under 200 words."
- "Write for a beginner."
- "Return a table with pros, cons, and risks."

Constraints should be specific enough to guide the answer but not so rigid that they block better alternatives.
